为什么要多个头？｜多头注意力机制（Multi-Head Attention）详解

在上一篇文章中我们梳理了单头 Self-Attention 的矩阵形式与缩放因子的来源。一个自然的问题是：既然一个注意力头已经能捕捉 token 之间的相似性，为什么要用多个头？本文来回答这个问题。

1. 从单头到多头：形式化定义

回顾单头注意力的形式：

$$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V) = \text{softmax}!\left(\frac{QW_i^Q (KW_i^K)^T}{\sqrt{d_k}}\right) VW_i^V$$

其中 $W_i^Q, W_i^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W_i^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$ 是第 $i$ 个头的投影矩阵。多头注意力的核心操作是：将 $h$ 个独立计算的注意力头沿最后一个维度拼接，再经一次线性投影回到原始维度：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \text{head}_2, \dots, \text{head}_h) , W_O$$

其中 $W_O \in \mathbb{R}^{h d_v \times d_{\text{model}}}$ 是输出投影矩阵。标准 Transformer 中取 $d_k = d_v = d_{\text{model}} / h$（例如原始论文中 $d_{\text{model}} = 512$，$h = 8$，则每个头的维度为 $d_k = d_v = 64$）。

关键区别：单头注意力的 $Q, K, V$ 各自只经过一组投影矩阵，而多头注意力为每个头分配了独立的三组投影矩阵 ${W_i^Q, W_i^K, W_i^V}$。这意味着不同头可以将输入映射到不同的表示子空间中进行注意力计算。

2. 为什么需要多个头？—— 从子空间的角度理解

如果只有一个头，注意力机制本质上是在单一语义空间中对 token 关系建模。考虑如下句子：

“The cat sat on the mat because it was tired.“

代词 “it” 与 “cat” 存在指代关系（共指消解），与 “sat” 存在主谓关系（句法依赖），与 “tired” 存在因果关系（语义推理）。单个注意力头需要在一个统一的表示空间中同时捕捉这三种性质迥异的关系——这无疑对 $d_k$ 维向量的表达能力提出了极高要求，且不同关系之间可能相互干扰。

多头的关键洞察在于：不同的头可以特化于不同层面的关系。

具体来说，每个头 $i$ 有自己独立的投影矩阵 $W_i^Q, W_i^K$，这意味着：

$$s_{ab}^{(i)} = \frac{(x_a W_i^Q)(x_b W_i^K)^T}{\sqrt{d_k}}$$

即 token $a$ 与 token $b$ 在第 $i$ 个头下的相似度分数，取决于它们在该头所定义的特定子空间中的距离。训练过程中，不同头会自发学习关注不同模式——有的头可能偏向句法依存，有的头可能偏向长距离语义关联，还有的头可能偏向位置邻近关系。这并非人为指定，而是通过梯度反向传播自然涌现的分工。

3. 为什么不直接增大单头的维度？

读者可能会问：如果目标是增加表达能力，为什么不直接把 $d_k$ 设大一些，而要拆成多个头再拼回来？

这是一个好问题。考虑两种方案的总参数量：

  • 单头大维度：$Q, K, V$ 投影参数共 $3 \times d_{\text{model}} \times (h d_k) = 3 d_{\text{model}} \cdot h d_k$
  • $h$ 头小维度：$h$ 组投影参数共 $h \times 3 \times d_{\text{model}} \times d_k = 3 d_{\text{model}} \cdot h d_k$

二者参数总量相同（加上 $W_O$ 也一样，因为输出维度固定为 $d_{\text{model}}$）。但在相同参数预算下，多头的优势在于：

归纳偏置（Inductive Bias）：多头的结构显式地迫使模型在 $h$ 个独立的低维子空间中分别计算注意力，而非在一个高维空间中”混”在一起。这种结构先验使得模型更容易学到解耦的表示——每个头只需在 $d_k = d_{\text{model}}/h$ 维的小空间里做好一件事，而不是在一个高维空间里同时做很多件事。

论文中的实验也印证了这一点：相同总维度下，$h > 1$ 的多头配置在翻译质量和收敛速度上均优于单头大维度配置。

4. 多头注意力的数学性质

我们考虑多头注意力输出的一个有趣性质。记第 $i$ 个头的注意力权重矩阵为 $A^{(i)} \in \mathbb{R}^{n \times n}$：

$$A^{(i)} = \text{softmax}!\left(\frac{QW_i^Q (KW_i^K)^T}{\sqrt{d_k}}\right)$$

那么该头的输出为 $\text{head}_i = A^{(i)} V W_i^V$，完整的多头输出为：

$$\text{MultiHead}(X) = \sum_{i=1}^{h} A^{(i)} , (V W_i^V) , W_{O}^{(i)}$$

这里 $W_O^{(i)} \in \mathbb{R}^{d_v \times d_{\text{model}}}$ 是 $W_O$ 按头切分后的第 $i$ 个分块，即 $W_O = [W_O^{(1)}; W_O^{(2)}; \dots; W_O^{(h)}]$。上式表明多头注意力的输出可以视为 $h$ 个独立注意力头的加权聚合（每一个头用自己的 $A^{(i)}$ 加权聚合 $V$ 在对应子空间中的映射），这种”各司其职、最后融合”的模式是多头机制的精髓。

另一个值得注意的是残差连接与多头的配合：Transformer 的每个 sublayer 采用 $\text{LayerNorm}(X + \text{Sublayer}(X))$ 结构。即便某个头学到了”无用”的注意力模式，残差连接保证了 $X$ 本身被原样保留，该头不会破坏已有信息——这为多头结构的稳定训练提供了重要保障。

5. 小结

多头注意力并非对单头注意力的简单重复，而是通过子空间划分这一结构性先验，使模型能够在相同参数预算下学到更丰富、更解耦的 token 表示。三个要点：

  1. 不同头将输入投影到不同的低维子空间中独立计算注意力；
  2. 相同参数量下，多头的归纳偏置使模型比”胖单头”更易训练、效果更好；
  3. 残差连接为多头结构的稳定性提供了底层保障——即使某些头暂时”失焦”，信息也不会被破坏。

多头结构的变体（MQA、GQA）和效率优化也是当前大模型推理加速的热点话题，我们在后续文章中继续探讨。
