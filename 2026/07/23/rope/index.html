从绝对到相对｜位置编码与旋转位置编码（RoPE）详解

在之前的文章中我们梳理了 Self-Attention 的核心机制，细心的读者可能已经发现一个盲点：注意力公式 $\text{softmax}(QK^T/\sqrt{d_k})V$ 中，所有 token 是同时、对称地参与运算的——公式本身对输入序列的排列顺序没有任何感知。也就是说，如果你把一句话的词序完全打乱，Self-Attention 给出的输出在数学上完全等价（只是行的顺序跟着变）。这显然不合理：”猫追老鼠”和”老鼠追猫”是两个不同的世界。如何让模型知道 token 之间的先后顺序？这便是位置编码（Positional Encoding）要解决的问题。本文从绝对位置编码出发，逐步过渡到当前大语言模型广泛采用的旋转位置编码（Rotary Position Embedding, RoPE），给出清晰的数学推导与直观理解。

1. 问题：为什么 Self-Attention 是置换不变的？

先做一个简单的论证。设输入序列为 $X = [x_1, x_2, \dots, x_n]^T \in \mathbb{R}^{n \times d}$，施加一个任意的行置换矩阵 $P \in {0,1}^{n\times n}$（置换矩阵的每一行每一列恰好有一个 1），得到打乱后的输入 $X’ = PX$。

此时注意力计算变为：

$$\text{Attention}(X’) = \text{softmax}!\left(\frac{(PX W_Q)(PX W_K)^T}{\sqrt{d_k}}\right)(PX W_V)$$

展开内积部分：

$$(PX W_Q)(PX W_K)^T = P X W_Q W_K^T X^T P^T = P \cdot \big(X W_Q W_K^T X^T\big) \cdot P^T$$

由于 softmax 是逐行运算（对矩阵的每一行独立做归一化），而行置换只改变行的顺序，因此整个 Attention 的输出等于原输出左乘 $P$——仅仅是行的顺序跟着变，每个 token 的输出表示完全不变。这意味着 Self-Attention 机制本身是一个”语义聚合器”，它负责回答 “哪些 token 之间有关系”，但完全不关心”这些 token 分别在什么位置”。

解决思路很自然：在输入进入 Attention 之前，把位置信息”注入”到 token 表示中。

2. 绝对位置编码：Sinusoidal Position Encoding

2.1 定义与直觉

原始 Transformer（Vaswani et al., 2017）的做法是：给每个位置 $pos$ 定义一个固定的向量 $\text{PE}(pos) \in \mathbb{R}^d$，直接加到 token embedding 上：

$$\tilde{x}_t = x_t + \text{PE}(t)$$

其中位置 $t$ 的编码向量的第 $i$ 个分量为：

$$
\text{PE}(t, 2i) = \sin!\left(\frac{t}{10000^{2i/d}}\right), \qquad
\text{PE}(t, 2i+1) = \cos!\left(\frac{t}{10000^{2i/d}}\right)
$$

这里 $d$ 是模型隐藏维度，$i = 0, 1, \dots, d/2 - 1$ 遍历每一对维度。直觉上，这像是在给每个位置分配一个不同频率的正弦波指纹——低频维度（$i$ 小）变化缓慢，编码”大局位置感”；高频维度（$i$ 大）变化剧烈，编码”精细位置感”。合并起来，每个位置获得了一个唯一的、$d$ 维的”坐标”。

2.2 为什么选三角函数？

一个自然的问题是：为什么不用最简单的可学习位置嵌入（Learnable Position Embedding），而是手工设计正弦函数？原因有两个层面。

第一，外推能力。可学习位置嵌入的维度是固定的——如果你训练时最大序列长度为 512，那位置 513 就没有对应的嵌入向量。而正弦函数定义在整个实数轴上，理论上可以为任意大的 $t$ 计算编码，具备天然的长度外推能力。

第二，相对位置的可线性表示——这是更深刻的性质。考虑相邻位置 $t$ 和 $t + \Delta$ 的编码之间的关系。对于第 $i$ 对维度，频率 $\omega_i = 1/10000^{2i/d}$，我们有：

$$\text{PE}(t + \Delta) = \big[\sin(\omega_i (t+\Delta)),; \cos(\omega_i (t+\Delta))\big]$$

利用三角恒等式：

$$\sin(\omega_i(t+\Delta)) = \sin(\omega_i t)\cos(\omega_i \Delta) + \cos(\omega_i t)\sin(\omega_i \Delta)$$
$$\cos(\omega_i(t+\Delta)) = \cos(\omega_i t)\cos(\omega_i \Delta) - \sin(\omega_i t)\sin(\omega_i \Delta)$$

写成矩阵形式：

$$
\begin{bmatrix}
\sin(\omega_i(t+\Delta)) \
\cos(\omega_i(t+\Delta))
\end{bmatrix}

\underbrace{\begin{bmatrix}
\cos(\omega_i \Delta) & \sin(\omega_i \Delta) \
-\sin(\omega_i \Delta) & \cos(\omega_i \Delta)
\end{bmatrix}}_{M_i(\Delta)}
\begin{bmatrix}
\sin(\omega_i t) \
\cos(\omega_i t)
\end{bmatrix}
$$

这意味着 $\text{PE}(t+\Delta)$ 可以通过一个仅依赖于 $\Delta$ 的线性变换从 $\text{PE}(t)$ 得到！换句话说，位置 $t$ 和 $t+\Delta$ 之间的几何关系被编码在一个与 $t$ 无关的变换矩阵中。这一性质正是后续所有相对位置编码工作的理论基石。

2.3 正弦编码的局限

尽管正弦位置编码在原始 Transformer 上是有效的，但它有一个结构性的缺点：位置信息是以”加法”的方式注入的，在经历多层变换后会被稀释。具体来说，输入层的 $\tilde{x}_t = x_t + \text{PE}(t)$ 进入 Attention 后，$QK^T$ 的计算展开为：

$$(x_m + \text{PE}(m))W_Q \cdot W_K^T (x_n + \text{PE}(n))^T$$

这会展开成交叉项 $x_m W_Q W_K^T \text{PE}(n)^T$ 和 $\text{PE}(m) W_Q W_K^T x_n^T$，但这些项混合了语义和位置，且依赖于绝对位置 $m$ 和 $n$——语义上相似但位置不同的两个 token 可能得到不同的注意力分数。这引出了一个重要的方向性转变：能不能让注意力分数只依赖于两个 token 之间的相对距离 $m - n$，而不是它们的绝对位置？

3. 相对位置编码的思想

在深入 RoPE 之前，我们先厘清”相对位置编码”究竟要达成什么。

设两个 token 的 query 向量为 $q_m$，key 向量为 $k_n$。在标准的绝对位置编码中，注意力分数为：

$$s(m, n) = q_m^T k_n$$

我们希望在计算注意力时，模型感知到的是 $m$ 和 $n$ 的相对偏移。不同的方法有不同的实现路径：

  • 显式相对位置偏置（如 T5, ALiBi）：直接在注意力分数上加减一个与 $m-n$ 有关的值；
  • 相对位置编码到 Key 中（如 Transformer-XL）：将相对位置向量直接编入 key 的计算；
  • 旋转位置编码（RoPE）：通过旋转 $q_m$ 和 $k_n$，使得它们的内积天然地只依赖于 $m-n$。

第三条路是最优雅的——它不修改注意力公式的结构，不引入额外参数，却巧妙地实现了相对位置编码。下面我们进入 RoPE 的核心推导。

4. 旋转位置编码（RoPE）

4.1 核心目标

RoPE（Su et al., 2023）的出发点是寻找一个函数 $f$，将位置信息融入 query 和 key 向量：

$$q_m = f_q(x_m, m), \quad k_n = f_k(x_n, n)$$

使得它们的内积只依赖于相对位置：

$$\langle f_q(x_m, m),; f_k(x_n, n) \rangle = g(x_m, x_n, m - n)$$

其中 $g$ 是某个函数，且其第三个参数只取 $(m-n)$。只要能满足这一点，注意力机制就自然具备了相对位置感知。

4.2 从二维旋转出发

先说结论：用二维旋转。考虑最简单的情况——$d=2$，此时 $q, k \in \mathbb{R}^2$。我们可以把向量逆时针旋转一个与位置成正比的角度：

$$f_q(x_m, m) = R(m\theta) \cdot x_m W_Q, \quad f_k(x_n, n) = R(n\theta) \cdot x_n W_K$$

其中 $R(\theta)$ 是标准的二维旋转矩阵：

$$
R(\theta) = \begin{bmatrix}
\cos\theta & -\sin\theta \
\sin\theta & \cos\theta
\end{bmatrix}
$$

这里 $\theta$ 是一个预设的基角度（类似正弦编码中的频率）。那么，两个旋转后向量的内积为：

$$\begin{aligned}
(R(m\theta), q)^T (R(n\theta), k) &= q^T R(m\theta)^T R(n\theta), k \
&= q^T R(-m\theta) R(n\theta), k \quad \text{（旋转矩阵的逆 = 旋转矩阵的转置）}\
&= q^T R((n-m)\theta), k
\end{aligned}$$

关键的第三步利用了旋转的可加性：先转 $-m\theta$ 再转 $n\theta$，等价于转 $(n-m)\theta$。而 $(n-m)\theta$ 正好只依赖于相对位置！于是：

$$\langle f_q(x_m, m), f_k(x_n, n) \rangle = q^T R((n-m)\theta) , k$$

这只与 $x_m, x_n$ 和相对位置 $n-m$ 有关，完美达成了我们的目标。这就是 RoPE 的几何直觉——用绝对位置的旋转，天然得到相对位置的表示。

4.3 推广到高维：分块旋转

二维的情况清楚了，但实际 $d$ 通常是几百维。怎么办？答案很直接：把 $d$ 维向量拆成 $d/2$ 个二维子空间，每个子空间用不同的旋转速度 $\theta_i$。具体地，定义一组频率：

$$\theta_i = 10000^{-2i/d}, \quad i = 0,1,\dots,d/2-1$$

对于 query 向量 $q \in \mathbb{R}^d$，将相邻两维 $(q_{2i}, q_{2i+1})$ 绕角度 $m\theta_i$ 旋转：

$$
\begin{bmatrix}
q_{2i}’ \[2pt]
q_{2i+1}’
\end{bmatrix}

\begin{bmatrix}
\cos(m\theta_i) & -\sin(m\theta_i) \
\sin(m\theta_i) & \cos(m\theta_i)
\end{bmatrix}
\begin{bmatrix}
q_{2i} \[2pt]
q_{2i+1}
\end{bmatrix}
$$

对 key 向量做同样的操作（角度为 $n\theta_i$），则对于第 $i$ 个二维子空间有：

$$(q_i’)^T k_i’ = q_i^T R((n-m)\theta_i) , k_i$$

将所有 $d/2$ 个子空间的内积累加，即可得到完整的内积 $\sum_{i=0}^{d/2-1} q_i^T R((n-m)\theta_i), k_i$。这个内积仍只依赖于相对位置 $n-m$，RoPE 在任意偶数维空间中都成立。

4.4 高效实现：复数视角

在实际代码中，逐个 2×2 矩阵乘法太低效。RoPE 有一个等价但更高效的实现方式，用复数乘法一次性完成所有维度的旋转。

将相邻两维视为复数的实部和虚部：$z = q_{2i} + j,q_{2i+1}$（这里用 $j$ 表示虚数单位以区别于下标 $i$）。二维旋转等价于乘以单位复数 $e^{j\theta}$：

$$z \cdot e^{j\theta} = (q_{2i} + j,q_{2i+1})(\cos\theta + j\sin\theta) = (q_{2i}\cos\theta - q_{2i+1}\sin\theta) + j,(q_{2i}\sin\theta + q_{2i+1}\cos\theta)$$

这与矩阵形式完全相同。推广到整个 $d$ 维向量，定义复数向量：

$$\vec{q} = [q_0 + j,q_1,; q_2 + j,q_3,; \dots,; q_{d-2} + j,q_{d-1}] \in \mathbb{C}^{d/2}$$

以及逐元素的复数旋转因子 $\Theta_m = [e^{j m\theta_0}, e^{j m\theta_1}, \dots, e^{j m\theta_{d/2-1}}]$。那么 RoPE 的作用等价于：

$$f_q(x_m, m) = \vec{q} \odot \Theta_m$$

其中 $\odot$ 表示逐元素（Hadamard）复数乘法。这个形式在现代 GPU 上可以高效实现：预先计算好 $\cos(m\theta_i)$ 和 $\sin(m\theta_i)$，然后用两组逐元素乘加即可完成旋转，无需显式构建任何旋转矩阵。实际工程中的参考实现如下（PyTorch 风格伪代码）：

5. RoPE 的优势：与其他方法的对比

至此，我们可以总结 RoPE 的几个核心优势：

（1）自然地实现相对位置编码，而不修改注意力公式。 RoPE 不改变 $\text{softmax}(QK^T/\sqrt{d_k})V$ 的结构，只是将 $Q$ 和 $K$ 在计算内积之前旋转了一下。注意力分数自动包含相对位置信息，无需像 T5 那样手动加偏置项，也无需像 Transformer-XL 那样重构 key 的计算方式。

（2）具备良好的外推能力。 RoPE 继承了正弦编码的优良基因——它使用的频率是 $10000^{-2i/d}$，高频维度（$i$ 大）旋转得快，对局部位置敏感；低频维度（$i$ 小）旋转得慢，对远距离仍然保持稳定。这使得 RoPE 在训练短序列后外推到更长序列时，性能衰减较慢。更进一步，已有大量工作（如 NTK-aware scaling、YaRN、ReRoPE 等）在 RoPE 的基础上做长度外推优化，形成了一个活跃的研究子领域。

（3）不引入额外参数。 与可学习位置嵌入相比，RoPE 的旋转角度是预设的（与正弦编码类似），训练时不需要为位置信息分配额外的参数。这在大模型时代尤为重要——减少参数意味着减少显存占用和通信开销。

（4）相对位置随距离自然衰减。 当 $|n-m|$ 较大时，不同频率的旋转角差异增大，各子空间的内积贡献趋于不相干叠加，注意力分数在统计意义上随距离衰减。这符合语言的局部性先验——一个词通常与附近的词关系更紧密。而这种衰减并非硬编码（不像 ALiBi 直接做线性减法），而是通过旋转操作的数学结构”软”体现的，为长距离依赖保留了建模空间。

（5）被实践充分验证。 从 LLaMA 系列（Llama、Llama 2、Llama 3）、Mistral、Qwen、DeepSeek 到 GPT-NeoX、Falcon 等主流开源大模型，RoPE 几乎已成为当前大语言模型位置编码的事实标准。一项技术被如此广泛的顶级团队独立采纳，本身就说明其设计的合理性。

6. 小结

本文从 Self-Attention 的置换不变性问题出发，系统梳理了位置编码的演进脉络：

  1. 正弦位置编码通过加法注入绝对位置，且天然携带相对位置的可线性表示性质，但位置信息在深层可能被稀释；
  2. 相对位置编码的思路是让注意力分数只依赖于 token 之间的相对距离，而非绝对坐标；
  3. RoPE 通过巧妙的旋转操作——用绝对位置的角度旋转 $Q$ 和 $K$，获得仅与相对位置有关的内积——优雅地统一了绝对编码和相对编码的优势；
  4. 复数视角下的逐元素旋转实现简洁高效，已成为当前大语言模型的主流选择。

理解 RoPE，本质上就是理解一个简单的几何事实：在二维平面上，先后旋转两个向量再做内积，其结果只由两个旋转角度的差决定。把这个二维的旋转铺到高维空间的每一对相邻维度上，就得到了旋转位置编码的全部。

位置编码这个看似细小的问题，折射出深度学习研究的一个经典范式：从发现结构缺陷（置换不变）出发，到提出粗糙但有效的补丁（正弦编码），再到深入思考”我们希望什么性质”（相对性），最终以一种数学上更优雅的方式重新实现该性质（旋转）。这种”问题驱动 → 性质定义 → 优雅重构”的演进路径，在 Self-Attention → Multi-Head Attention → RoPE 这条线上一再重演，也是理解现代 Transformer 架构设计最好的思维框架。
